# N8: 完成

所有 feature 的所有任务完成后：

## 进入消息

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🏁 N8 收尾 — 全部 Feature 完成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 1. 调用 tc-doc-syncer

检查 `tc-doc-syncer` 是否在可用 skill 列表中：

- **可用** → 调用执行：
  - README 精炼更新（架构 + 业务 + 快速开始）
  - .claude/CLAUDE.md 和 rules/ 同步
  - specs CHANGELOG 按日期生成
  - 文档一致性验证
- **不可用** → 跳过并明确输出，**不阻塞总结**

**输出**（强制，二选一）：

```text
📝 文档同步: 调用 tc-doc-syncer...
   ✓ README ({N} 个项目) | ✓ CLAUDE.md | ✓ rules/ | ✓ CHANGELOG
```

或

```text
📝 文档同步: ⏭ 跳过（tc-doc-syncer 未注册）
```

## 2. 输出总结（强制）

```text
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎉 /tc-ai 全部完成
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📂 Features: {完成数}/{总数}
📋 总任务: {完成数}/{总数}
⏱  总耗时: {时间}
📝 文档同步: {已完成 / 跳过}

各 Feature 摘要:
- 1.{name}: {N} 个任务 ✅ ({耗时})
- 2.{name}: {N} 个任务 ✅ ({耗时})
- ...

📌 后续建议:
- 检查变更文件: git status
- 提交前跑一次完整构建/测试
- 如有 LESSONS.md 更新，团队同步一下
```
