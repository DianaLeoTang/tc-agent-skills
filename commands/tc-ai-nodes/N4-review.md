# N4: Review

每个 task 完成后必须执行，按**单个 task 粒度**审查。

## 进入消息

```text
🔍 N4 Review — Task {T-编号}
```

## 1. AI 自审

检查本 task 变更的：

- 代码质量：命名、结构、可读性、是否符合 `.claude/rules/`
- 逻辑正确性：边界条件、错误处理、并发安全
- 安全性：硬编码密钥、`.env` 误入 git、注入漏洞、OWASP Top 10
- 性能：N+1 查询、不必要的重复计算、内存泄漏风险

发现问题立即修复，不确定则暂停。

**输出自审结果**（强制）：

```text
🔍 AI 自审:
   ✓ 代码质量 / 逻辑 / 安全 / 性能
   {发现问题时:}
   ⚠ 发现 {N} 个问题:
     - {问题1} → {已修复 / 待用户确认}
     - {问题2} → ...
```

## 2. 外部 Review（如 `codex:rescue` 可用）

检查 `codex:rescue` 是否在可用 skill 列表中：

- **可用** → 传入本 task 的 diff，按下面规则处理：
  - 合理建议 → 修复后重新提交复审
  - 误报 → 记录理由后忽略
- **不可用** → 跳过本步并明确输出，**不阻塞流程**

**输出外部 Review 结果**（强制，二选一）：

```text
🤖 Codex Review:
   ✓ 通过 / ⚠ {N} 条建议（已修复 {N} 个，忽略 {N} 个）
```

或

```text
🤖 Codex Review: ⏭ 跳过（codex:rescue 未注册）
```

## 退出消息（强制输出）

```text
✅ N4 Review 通过 → 进入 N5 标记完成
```

发现严重问题且自动修复失败 → **暂停并报告**，不继续 N5。
